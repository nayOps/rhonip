MorphoTablet Customization Package Guide

Customization package tools

MTabletCustoPackTool is a Linux tool which generates a customization package to customize MorphoTablet firmware.

Description

MTabletCustoPackTool can generate a customization package from "integration" folder.
It can also generate an integrator key set, which is needed to sign customization packages and system applications.
Each integrator key generated shall be signed by IDEMIA.

Both tool and "integration" folder are located here.

Command line

$ ./MTabletCustoPackTool OPTIONS

OPTIONS:
-k, --generate-key-set KEYNAME generate a customization package key set named KEYNAME, as well associated Android Studio keystore
-s, --system-sign-apk KEYNAME INAPKFILE OUTAPKFILE provide system permission to INAPKFILE using KEYNAME keyset
-e, --export-unsigned-public-key KEYNAME OUTFILE export OUTFILE to provide KEYNAME public key to MorphoTablet™ manufacturer for signature
-i, --import-signed-public-key KEYNAME INFILE import INFILE from MorphoTablet™ manufacturer to use as KEYNAME public signed key
-p, --generate-customization-package KEYNAME OUTZIP [OPTIONS2] generate a ready to install customization package OUTZIP using KEYNAME key set
-h, --help display this help

OPTIONS2:
-a, --alternative-key-set ALTKEYNAME generate a customization package allowing further update by KEYNAME or ALTKEYNAME signed customization packages
-v, --customization-package-version VERSION set customization package version (0-4294967295, 0 by default)
-d, --downgrade-policy POLICY after installing generated customization package, next update will be impacted by downgrade POLICY, see below
-x, --key-check-disable after flashing generated customization package, any integrator key will be accepted

POLICY:
DOWNGRADE_ALLOWED No restriction (by default)
NO_VERSION_DOWNGRADE Customization package will refuse inferior version updates
NO_DATE_DOWNGRADE Customization package will refuse inferior generation date updates
NO_DATE_NOR_VERSION_DOWNGRADE Enable both NO_VERSION_DOWNGRADE and NO_DATE_DOWNGRADE usual process

  • 1. Generating integrator keys:
    • $ ./MTabletCustoPackTool --generate-key-set mycompany1
    WARNING:
  • Please keep generated keyset in a safe place, since it allows full access to tablet.
  • Do not loose this keyset, since it might be the only way to update / remove a flashed customization package.


  • 2. Signing integrator public keys:
    • $ ./MTabletCustoPackTool --export-unsigned-public-key mycompany1 mycompany1.bin
    Note:
  • Please provide "mycompany1.bin" to Idemia and get "mycompany1_signed.bin" back before proceeding ...
    • $ ./MTabletCustoPackTool --import-signed-public-key mycompany1 mycompany1_signed.bin
  • 3. Generating integrator customization package:
    • $ ./MTabletCustoPackTool --generate-customization-package mycompany1 mycustpack_update.zip
    WARNING:
  • Flashing a Customization Package will lock tablet to customization package key. Tablet won't accept customization packages generated with another key.
  • Key can be changed by using --alternative-key-set option or --key-check-disable option. Please note that current flashed key is needed to do so.
  • Please note that --key-check-disable option will create a customization package that will allow anyone to flash any new key. Please keep such generated customization packages in a safe place or use this option with unsecure keys used for demo or for developpers for example.


  • 4. Importing customization package in MorphoTablet™ 2(i):
    • Please refer to Importing customization package in MorphoTablet™ 2(i) chapter for more details
