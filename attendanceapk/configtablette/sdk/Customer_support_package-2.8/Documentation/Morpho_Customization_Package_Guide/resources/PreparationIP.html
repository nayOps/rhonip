MorphoTablet Customization Package Guide

Prepare a customization package

Populate pre-defined file tree

To prepare a customization package, you first need to populate a pre-defined file tree with your third-party applications.
Current file tree is located here and contains four important folders: "app", "priv-app", "preinstall" and "etc/security".
Each folder has a specific role:

  • Applications copied to integration/app won't have system privilegies and won't be removable by the user.
  • Applications copied to integration/preinstall won't have system privilegies and will be removable by the user.
  • Applications copied to integration/priv-app will have system privilegies and won't be removable by the user.
  • /etc/security is used to verify OTA update signature

Important notes:

  • It is recommended to copy each apk application in a specific folder, for example priv-app/myapp/myapp.apk. Indeed, if your application uses native libraries, customization package tool will extract them beside application, and if two application have same library file name, only one will be kept.
  • For Android firmware to provide third-party applications system permissions, a reference package called "com.morpho.integrator.refsignature" that is signed with third-party company platform key is present by default.

Once integration folder is complete, it's time to generate the customization package with MTabletCustoPackTool.
