Debugging signed applications with Android Studio

Description

When debugging applications requesting specific rights, customization package generated key can be used by Android Studio to sign applications when running them.

Prerequisite :

  • A customization package key (signed by Idemia) which we will name "mykeyname" in this example.
  • A customization package generated with "mykeyname".
  • A tablet flashed with previously generated customization package.

    • Configuring Android Studio :

  • 1. Open customization package generated key directory:

    • Please look at your customization Package generated key directory, it contains following files :

  • Android_studio_keystore.jks : This is keystore file to provide to Android Studio.
  • Android_studio_keystore.txt : This text files contains parameters to configure Android Studio.
  • integratorkey
  • integratorkey.pub
  • integratorkey.pub.sign
  • platform.pk8
  • platform.x509.pem
  • releasekey.pk8
  • releasekey.x509.pem

  • 2. Read keystore settings :
    • $ cat mykeyname/Android_studio_keystore.txt
Added key : alias = mykeyname, keypass = mykeypassword   , keystorepass = mystorepassword

  • 3. Open your Android Studio project and go to project settings and create a signing configuration :



  • 4. Select created signing configuration for your current build :



  • You can now run and debug signed applications directly from Android Studio.
