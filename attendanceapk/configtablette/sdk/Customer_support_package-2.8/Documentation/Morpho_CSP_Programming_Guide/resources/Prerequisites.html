Android Debug Bridge (ADB) Setup

To allow your PC to communicate with the tablet, you first need to enable the Android Debug Bridge following those instructions: https://developer.android.com/studio/command-line/adb.html#Enabling.

On Windows, you will typically have to install an extra driver to be able to communicate with the MorphoTablet through ADB.
As a workaround to a software bug, you first have to enable the MTP mode on your MorphoTablet to make it expose the USB device expected by the driver.
When the tablet is connected, slide down the status bar to access USB options, and enable MTP.

Then you can install the ADB driver that is available for Windows 7 and Windows 10 in the ADB_Drivers directory.
On other Windows versions, you might see a security warning that you can ignore during driver installation.

Note: If ADB is not detecting the tablet, please try to create a file "adb_usb.ini" in your ".android" directory (~/.android on Linux, C:\Users\YOUR_LOGIN\.android on Windows) with a single line mentioning the USB vendor ID of the tablet: 0x225D. On Linux, you will need to restart the adb daemon.
