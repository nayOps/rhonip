Understanding PeripheralPowerManagement Sample Application

Click to View Source

A developer can turn on power for a particular hardware peripheral like SAM card, Smart Card, CBM, Usb host, etc... For example if an application is accessing only smart card and CBM then developers can turn off other hardware peripheral devices. If they do so, The tablet performance will be improved saving battery.

Functions

In order to use any hardware peripheral "get/set" feature developers need to add the two following permissions to the manifest file:

<uses-permission android:name="android.permission.READ_SETTINGS">
<uses-permission android:name="android.permission.WRITE_SETTINGS">

Import org.simalliance.openmobileapi.jar into sample application in the libs folder.

II) Method available in software TA0 and higher (recommended)

Description – The following methods are available in software TA0 and higher.

A) Get the hardware peripheral status

Description – Below API is used to know the hardware peripheral status.

Parameters:
ContentResolver cr - ContentResolver.
String name - string to select a device, example:"device_more_otg_enabled".

//Object of class Settings.System has method getInt(ContentResolver cr, String name),
// this is the main function that we are using in our sample to get the hardware peripheral status
int valueOTG = Settings.System.getInt(getContentResolver(), DEVICE_MORE_OTG_ENABLED);

B) Swith On/OFF devices

Description – Below API is used to give power to different peripheral components of the tablet, like USB, Fingerprint sensor or Smart Card.

Parameters:
ContentResolver cr - ContentResolver.
String name - string to select a device, example:"device_more_otg_enabled".
int value - one to power on the device or zero to power off.

//Object of class Settings.System has method putInt(ContentResolver cr, String name, int value),
//this is the main function that we are using in our sample to switch on or off devices
Settings.System.putInt(getContentResolver(), DEVICE_MORE_OTG_ENABLED, value);

III) Deprecated methods

Description – The following methods are deprecated started at software TA0 and higher.

A) Turn ON/OFF SAM card reader

Description – The below function is used to power on the SAM reader.

private SEService seService = null;
private void setSamCardPower(boolean on) {

    try {
        if (seService == null) {
            SEService.CallBack callback = new SEService.CallBack() {
                public void serviceConnected(SEService service) {
                    seService = service;
                }
            };
            seService = new SEService(this.getApplicationContext(),
                    callback);
            do {
                Object mWaitOMACheckCert = new Object();
                try {
                    synchronized (mWaitOMACheckCert) {
                        mWaitOMACheckCert.wait(1000);
                        Log.d("debug", "wait 1s for OMA check.");
                    }
                } catch (InterruptedException e) {
                    Log.d("debug", "interruped.");
                }
            } while (seService == null);
        }

        Reader[] readers;
        readers = seService.getReaders();
        Reader reader = readers[3];
        if (on) {
            Log.d("PowerManagement", "readers[3].power_on_reader()");
            reader.power_on_reader();
        } else {
            Log.d("PowerManagement", "readers[3].power_off_reader()");
            reader.power_off_reader();
        }

    } catch (Exception e) {
        e.printStackTrace();
    }
}
