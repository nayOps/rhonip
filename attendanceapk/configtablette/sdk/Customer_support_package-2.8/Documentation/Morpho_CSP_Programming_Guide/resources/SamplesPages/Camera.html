The device has two cameras, one 2.0 MP front camera and one 13.0 MP rear camera. In order to use the camera, it's needed for the users to get a feedback of what the camera sees. This document will be split into two chapters:

    - The Preview class : visual feedback
    - The camera control : camera access

In order to use any camera feature, developers need to add the following lines to the manifest: 

<uses-feature android:name="android.hardware.camera">
<uses-permission android:name="android.permission.CAMERA">
<uses-feature android:name="android.hardware.camera.autofocus" android:required="true">

I) Preview Class

Description - The preview class needs to implement SurfaceHolder.callback in order to capture the callback events for creating and destroying a view.

public class Preview extends ViewGroup implements SurfaceHolder.Callback {
  ...
}

Description - The following method initializes the Preview class. 

public void init(Context context){
    SurfaceView surfaceView = new SurfaceView(context);
    addView(surfaceView);

    // Install a SurfaceHolder.Callback so we get notified when the
    // underlying surface is created and destroyed.
    mHolder = surfaceView.getHolder();
    mHolder.addCallback(this);
    // mHolder.setType(SurfaceHolder.SURFACE_TYPE_PUSH_BUFFERS);
}

Description - The following method sets the camera that will be displayed on screen. 

public void setCamera(Camera camera,CaptureImageActivity cia) {
    mCamera = camera;
    if (mCamera != null) {
        mSupportedPreviewSizes = mCamera.getParameters().getSupportedPreviewSizes();
        setCameraDisplayOrientation(cia,cia.cameraCurrentlyLocked, mCamera);
        requestLayout();
    }
}

Description - The following method reconfigures the preview according to the input camera.

public void switchCamera(Camera camera) {
    try {
        camera.setPreviewDisplay(mHolder);
    } catch (IOException exception) {
        Log.e(TAG, "IOException caused by setPreviewDisplay()", exception);
    }
    Camera.Parameters parameters = camera.getParameters();
    parameters.setPreviewSize(mPreviewSize.width, mPreviewSize.height);

    requestLayout();

    camera.setParameters(parameters);
}

II) Camera control

Description - How to take a picture

btnCapture.setOnClickListener(new OnClickListener() {
    @Override
    public void onClick(View v) {

        if (btnCapture.getText().equals("Resume")) {
            camera.startPreview();
            safeToTakePicture = true;
            btnCapture.setText("Click");
        } else {
            if (safeToTakePicture) { // Checking if ready to take next
                // picture or not.

                Parameters parameters = camera.getParameters();

                if (cameraCurrentlyLocked == 1)
                    parameters.setPictureSize(1600, 1200);
                else
                    parameters.setPictureSize(2560, 1920);
                parameters.setRotation(rotation());
                camera.setParameters(parameters);
                camera.takePicture(shutterCallback, rawCallback,
                        jpegCallback);
                safeToTakePicture = false;
            }
        }
    }
});

Description - jpegCallback saves the taken picture.

PictureCallback jpegCallback = new PictureCallback() {
    @Override
    public void onPictureTaken(byte[] data, Camera camera) {
        FileOutputStream outStream = null;
        try {
            // Write to SD Card

            String fileName = String.format(
                    Environment.getExternalStorageDirectory().getPath()
                            + "/mv_%d.jpg", System.currentTimeMillis());
            outStream = new FileOutputStream(fileName);
            outStream.write(data);
            outStream.close();

            resetCam();
            Toast.makeText(getApplicationContext(), "Saved at " + fileName,
                    Toast.LENGTH_SHORT).show();

            showCapturedImage(fileName);

        } catch (FileNotFoundException e) {
            Log.e(this.getClass().getName(), "PictureCallback", e);
        } catch (IOException e) {
            Log.e(this.getClass().getName(), "PictureCallback", e);
            e.printStackTrace();
        }
        safeToTakePicture = true;
    }
};

III) Android life cycle

Description - camera should be released and reset according to the android lifecycle

protected void onResume() {
  cameraCurrentlyLocked=sPrefrences.getInt("cid",0);
  camera = Camera.open(cameraCurrentlyLocked);
  preview.setCamera(camera, this);
  camera.startPreview();
}
protected void onPause() {
  if (camera != null) {
    camera.stopPreview();
    preview.setCamera(null, this);
    camera.release();
    camera = null;
  }
  super.onPause();
}
